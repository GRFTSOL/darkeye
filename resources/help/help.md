## 三、管理模块

管理模块负责对作品、标签、制作商等基础数据进行维护，并提供批量操作与导出。

### 1. 添加 / 修改作品

- 入口：`管理 -> 添加/修改作品`
- 主要功能：
  - 手动添加新作品：填入番号、标题、制作商、女优、男优、标签等信息。
  - 修改已存在作品：修正爬虫信息、补充缺失字段。
  - 爬虫辅助修改信息
  - 修改变色提醒


### 2. 作品标签管理

- 入口：`管理 -> 作品标签管理`
- 主要功能：
  - 添加、修改、删除标签(删除标签需要这个标签不存在于任何的作品中)。
  - 给标签设置说明，用于后续筛选和分析。
  - 未来会逐步配合“tag 计算器 / 高级过滤”功能增强（见 `README.md` 中 roadmap）。

### 3. 番号制作商管理

- 入口：`管理 -> 番号制作商管理`
- 主要功能：
  - 维护制作商 / 厂牌列表。
  - 对自动识别不准的制作商进行手动补全或修正。
  - 对无法从爬虫中直接获得制作商信息的番号进行补录。

### 4. 批量操作

- 入口：`管理 -> 批量操作`
- 当前常用能力：
  - **查找本地视频并录入添加**：
    - 前提：在 `设置 -> 视频` 中添加了本地视频所在路径。
    - 功能：扫描本地视频文件，并尝试根据文件名中的番号，自动在数据库中创建/关联作品。
    - 注意：
      - 会配合爬虫慢慢启动爬取信息，默认约 **20s 一次请求**，速度较慢。
      - 当前识别准确度有限，可能需要手动修正。
  - 其他批量操作（如批量更新字段、批量打标签等）会在后续版本中逐步增强。

### 5. 汇总查询表

- 入口：`管理 -> 汇总查询表`
- 主要用途：
  - 对作品数据进行汇总查看与筛选。
  - 提供**导出功能**，目前主要用于导出为 `.csv` 等格式，方便在 Excel / 其他工具中继续分析。

### 6. 综合管理

- 用于集中查看与维护多类数据（作品 / 演员 / 标签等）。
- 后续版本会继续补充更多高级管理视图（例如：更复杂的过滤、批量编辑等）。

### 7. 回收站

- 删除作品、标签等数据后会进入回收站。
- 可以在这里进行：
  - 恢复误删的数据。
  - 彻底删除不再需要的记录。

## 采集与浏览器插件

目前主要有三种方式采集/收藏作品数据：

1. **浏览器插件采集（推荐）**
   - 安装 Firefox 插件（详见 `README.md` 中“插件安装”章节）。
   - 启动软件后，打开 `javdb` / `javlibrary` / `javtxt` 等网站。
   - 在页面中点击插件提供的“收藏/收录”按钮开始采集。
   - 采集时会弹出 Firefox 窗口：
     - 访问 `javlibrary` 第一次需要手动过一次 Cloudflare 盾，大约能维持约 20 分钟。

2. **从本地视频反向识别**
   - 在 `设置 -> 视频` 中配置本地视频目录。
   - 打开 `管理 -> 批量操作 -> 查找本地视频并录入添加`。
   - 程序会：
     - 扫描本地视频文件。
     - 按文件名推断番号。
     - 将结果加入爬虫列表，定时（约 20s 一次）尝试自动爬取信息。
   - 提示：目前识别与匹配仍然比较不准，建议后续在管理界面中进行人工核对与修正。

3. **手动添加番号**
   - 任何时候都可以使用快捷键 **`W`** 直接打开“快速添加作品”窗口。
   - 手动输入番号与关键信息，适合针对性补录或特殊需求。

### 女优数据采集与更新

- 女优信息自动化采集目前**准确度有限**：
  - 右键点击女优栏进入编辑界面，可以点击“爬虫”按钮尝试自动更新。
  - 此方式仅适用于**无同名女优**的情况。
  - 对于重名女优，建议通过浏览器插件手动选择正确的条目再更新。

## 备份与迁移（版本更新相关）

软件升级或更换版本时，建议**先做好备份**，避免数据丢失。当前有两种方式：

1. **手动迁移（旧版本 / 无迁移工具时）**
   - 直接复制旧版本中的：
     - `resources/public`
     - `resources/private`
   - 覆盖到新版本的同名目录，即可沿用原有数据库。

2. **使用内置备份/还原工具（新版本推荐方式）**
   - 在软件中使用：
     - “备份私库” / “备份公库” 功能，将数据库导出到你选择的位置。
   - 在新版本中：
     - 使用“还原”功能，选择之前备份生成的 `meta.json` 和对应的 `.db` 文件。
     - 完成还原后**重启软件**。
   - 说明：目前仍无法做到完全无感迁移，部分情况下可能会有小问题，遇到异常可手动检查 `resources/public` / `resources/private` 中的数据文件。

> 更多关于数据库结构、安全与隐私的说明，请参考 [Security](docs/security.md)。
