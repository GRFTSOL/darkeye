# 爬虫并发崩溃恢复清单

适用范围：`core/crawler/CrawlerManager.py` 多任务并发抓取时出现崩溃/卡死/结果串线。

## P0（先止血，优先当天完成）

- [ ] 增加 `serial` 去重与在途任务去重（队列 + 活动任务 + `tasks` 三层去重）。
- [ ] 禁止同一 `serial` 重入覆盖 `self.tasks[serial]`。
- [ ] 修复临时图片命名冲突，改为全局唯一名（`uuid` 或 毫秒时间戳 + 随机串）。
- [ ] 停止使用 `id(worker)` 作为 `_relays` key，统一使用稳定业务键。
- [ ] 在任务完成/失败/超时后统一清理 `tasks`、`_relays`，避免状态泄漏。
- [ ] 给 `Worker` 异常增加完整堆栈日志（当前只记录异常字符串，不足以定位）。

## P1（稳定性加固，1-2天完成）

- [ ] 增加并发上限（总任务并发 + 单来源并发），防止线程池被阻塞任务耗尽。
- [ ] `javlib` 回传链路增加“请求唯一ID”关联，避免多请求复用同一信号通道串线。
- [ ] 对下载流程增加原子写入（先写临时文件再 `rename`），避免并发覆盖和脏文件。
- [ ] 为数据库写入增加冲突兜底（重复插入、锁等待、重试策略）。
- [ ] 增加任务状态机（queued/running/merging/downloading/done/failed）用于可观测性。

## P2（工程化完善，本周内完成）

- [ ] 增加并发压力测试（10/30/50 个 serial 批量，重复 serial 混入场景）。
- [ ] 增加故障注入测试（网络超时、源站空结果、下载失败、DB busy）。
- [ ] 增加运行期指标：`activeThreadCount`、队列长度、失败率、平均耗时。
- [ ] 增加崩溃后自动恢复策略（失败重试、任务重放、失败清单导出）。

## 验收标准（完成判定）

- [ ] 连续 3 轮批量抓取（每轮 >= 30 条）无崩溃、无假死。
- [ ] 日志中无任务串线（A 的结果写到 B）和重复 merge。
- [ ] 无临时文件冲突、无图片覆盖错乱。
- [ ] 无 `database is locked` 导致的任务终止。
- [ ] 失败任务可重试且不会污染后续任务状态。

## 回滚与应急

- [ ] 保留修复前分支/tag，支持一键回退。
- [ ] 若线上再次崩溃：先降并发 -> 暂停批量入口 -> 导出失败队列 -> 分批重跑。
- [ ] 保留最近一次崩溃日志与请求样本，用于二次复盘。
