    行为准则（Code of Conduct）¶

    本行为准则适用于参与 DarkEye 项目的所有贡献者与用户，包括但不限于在本仓库中的 issue、Pull Request、讨论区以及与本项目相关的任何社区空间。

    我们的承诺¶

    我们承诺为所有参与者提供一个无骚扰、友好且包容的环境，使每个人都能在不受歧视和冒犯的前提下进行协作与交流。

    无论你的经验水平、性别、性取向、残障、外貌、体型、种族、年龄、宗教或技术偏好如何，你都应被尊重。

    我们的标准¶

    有助于营造正向环境的行为包括但不限于：

    • 友善与尊重：以礼貌、尊重的方式与他人交流，关注问题本身而非针对个人。
    • 建设性反馈：提出清晰、具体的改进建议，而非人身攻击。
    • 主动帮助：在力所能及的范围内帮助新人或遇到困难的贡献者。
    • 尊重差异：理解并尊重不同的观点、经验和使用场景。

    不可接受的行为包括但不限于：

    • 骚扰性或侮辱性言论，包括性别、性取向、种族、宗教、外貌等方面的攻击。
    • 恐吓、威胁或跟踪行为。
    • 公开他人的隐私信息（如地址、真实姓名未公开信息、联系方式等）而未获得明确授权。
    • 持续性骚扰，包括在被明确要求停止后仍继续的行为。
    • 在专业环境中可被合理认为不适当的其他行为。

    适用范围¶

    本行为准则适用于与 DarkEye 项目相关的所有场景，包括但不限于：

    • 本 GitHub 仓库内的所有活动（issue、Pull Request、评论、讨论等）；
    • 与项目相关的聊天群组、论坛或其他线上社区；
    • 与项目直接相关的线下交流或活动。

    违纪处理¶

    发现违反本行为准则的行为时，项目维护者可以根据具体情况采取包括但不限于以下措施：

    • 私下口头或书面警告；
    • 要求公开或私下为不当行为道歉；
    • 暂时或永久禁止参与本项目的某些活动（如关闭或锁定 issue/PR）；
    • 在严重情况下，将相关行为报告给所在平台或其他有权限的机构。

    项目维护者有权删除、编辑或拒绝与本行为准则不符的评论、提交、代码、Wiki 编辑、issue 或其他贡献。

    举报方式¶

    如果你认为有人在本项目相关的空间中违反了本行为准则，或者你亲身经历了不当行为，请尽可能不要在公开渠道直接披露细节，而是通过以下方式联系项目维护者：

    • 邮箱：security@example.com（请根据实际情况替换）

    为便于调查和处理，请在举报中尽量提供以下信息：

    • 事件发生时间与大致地点（例如某个 issue/PR 链接）；
    • 相关参与者的用户名或可识别信息；
    • 你认为不当的具体行为描述；
    • 如有必要，请附上截图或日志截图（注意脱敏隐私信息）。

    我们会尽力在合理时间内回复并评估事件，并在尊重当事人意愿的前提下采取适当措施。

    与安全问题的关系¶

    有关安全漏洞或隐私风险的报告，请优先参考 docs/security.md 中的安全策略（Security Policy）与报告安全问题章节。
    行为准则侧重的是社区与沟通行为，而安全策略侧重的是软件本身的安全性与数据隐私。
